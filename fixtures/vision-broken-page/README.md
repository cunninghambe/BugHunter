# vision-broken-page fixtures

These PNG files are minimal 1×1 pixel images used as static fixtures for vision unit and integration tests.

- `broken-layout.png` — fixture representing a page with a visual layout defect. Used in tests with a stubbed VisionClient that returns a canned "broken layout" response.
- `clean-page.png` — fixture representing a visually clean page. Used in tests with a stubbed VisionClient that returns `{"anomalies": []}`.

## Regenerating

Both files are minimal valid PNGs created with Node:

```js
const png1x1 = Buffer.from([
  0x89,0x50,0x4e,0x47,0x0d,0x0a,0x1a,0x0a,
  0x00,0x00,0x00,0x0d,0x49,0x48,0x44,0x52,0x00,0x00,0x00,0x01,0x00,0x00,0x00,0x01,
  0x08,0x02,0x00,0x00,0x00,0x90,0x77,0x53,0xde,
  0x00,0x00,0x00,0x0c,0x49,0x44,0x41,0x54,0x08,0xd7,0x63,0xf8,0xcf,0xc0,0x00,0x00,
  0x00,0x02,0x00,0x01,0xe2,0x21,0xbc,0x33,
  0x00,0x00,0x00,0x00,0x49,0x45,0x4e,0x44,0xae,0x42,0x60,0x82,
]);
require('fs').writeFileSync('broken-layout.png', png1x1);
require('fs').writeFileSync('clean-page.png', png1x1);
```

The stub VisionClient in `vision-integration.test.ts` maps screenshot SHA-256 → canned response, so the actual pixel content does not matter. If you replace these with real screenshots, update the stub map in the integration test accordingly.
